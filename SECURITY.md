# Security Policy

## Reporting a vulnerability

Please report security issues privately. Do **not** open a public GitHub issue.

Preferred: open a private GitHub Security Advisory for this repository.

Include as much of the following as possible:

- Affected versions/commits
- Reproduction steps or proof of concept
- Impact assessment
- Any suggested mitigations

Please avoid including real secrets (tokens, API keys) in reports. If a secret is involved, rotate
it immediately and include only redacted values.

We aim to acknowledge reports within 72 hours and will coordinate a fix and disclosure timeline
with the reporter.

## Supported versions

We currently provide security fixes for:

- The `main` branch
- The latest tagged release (when releases exist)

Older versions may not receive security patches; upgrade to the latest supported version.

## Disclosure policy

We follow coordinated disclosure. If you would like attribution in a release
note or advisory, please include your preferred name/handle.
